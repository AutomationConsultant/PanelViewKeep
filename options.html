Options

Changelog

1.5

Added more options to the context menu:

  • Right-click on anything: Create new note
  • Right-click on selected text: Copy text to Keep
  • Right-click on a link: Copy link to Keep
  • Right-click on an image: Copy image path to Keep

Please note that the context menu additions are experimental. The new options don't work when in panel mode; this is a limitation of the Chrome API and I'm hoping that this bug will be fixed in a future version.

Added an option to disable the context menu.

1.4

Added context menu back in.

1.3

Removed context menu.

Added options page. Choose how Keep opens - in a panel, popup, or tab - and how large the panel or popup will be.

Link to options page appears in Keep! In the desktop site, in the gear menu; in the mobile site, next to the view buttons.

1.2

Clicking the extension icon switches to open instance of Keep (whether it's open in a normal tab window or the extension's panel) instead of opening a new window every time.

1.1

Changed to open Keep in a panel by default.

1.0

The first version.

Known Issues

File uploads don't work in Keep when viewing in the panel. This is a Chrome issue.

The panel does not always honor width and height dimensions if they are high values, even though the popup does.

Opening the options page will not stay in the same window when in panel mode, so I've made the mobile version's options link always open in a new tab.

The Create new note and Copy to Keep context menu options do not work when Keep is in a panel. This is a Chrome issue due to the API not being able to see the panel window's tab ID.

Have a problem I haven't found yet? Let me know on the support page.
